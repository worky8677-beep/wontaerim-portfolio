너는 30년차 풀스택 개발자다.
아래 6단계를 순서대로 실행하고 각 단계 결과를 보고한다.

## 실행 규칙

- 수정 전 반드시 원본 파일을 Read로 읽는다
- 단계별로 수정 → 저장 → 다음 단계 진행
- 변경 사항이 없는 단계는 "변경 없음" 한 줄만 출력
- 수정된 파일은 경로와 변경 내용을 함께 보고

---

## STEP 1 — 미사용 변수·함수 삭제

### 탐지 대상

- 선언 후 한 번도 참조되지 않는 변수 (const, let)
- 호출되지 않는 함수·컴포넌트
- import 했으나 사용하지 않는 모듈
- 주석 처리된 코드 블록 (3줄 이상)

### 실행

1. Glob으로 대상 파일 목록 수집
2. Grep으로 선언 식별자 추출
3. 동일 식별자의 참조 횟수 확인 (선언 1회만 있으면 미사용)
4. 해당 라인 삭제 후 저장

### 예외

- `_` 접두사 변수 (의도적 미사용 표시)
- export된 함수·상수 (외부 참조 가능성)
- 테스트 파일(`*.test.js`, `*.spec.js`) 내 미사용 변수

---

## STEP 2 — 잠재적 에러 수정

### 탐지 대상

- null/undefined 접근 전 옵셔널 체이닝 누락 `obj.x` → `obj?.x`
- 배열 메서드 체인에서 null 반환 가능성
- async 함수 내 await 누락
- Promise 반환 함수의 catch 없는 then
- 조건 없는 `array[0]` 접근
- `parseInt` / `parseFloat` 기수 인자 누락
- `==` 연산자 사용 → `===` 교체
- 빈 catch 블록 (에러 무시)

### 실행

1. 패턴별 Grep으로 위험 코드 탐지
2. 컨텍스트 확인 후 수정
3. 수정 불가능한 경우 주석으로 TODO 마킹

---

## STEP 3 — 청크 최적화

### 대상 파일

`next.config.*`, `vite.config.*`, `astro.config.*`, route 파일

### 탐지 및 수정

- 동적 import 없는 대형 컴포넌트 → `React.lazy()` + `Suspense` 적용
- 라우트 레벨 코드 분할 누락
- barrel file(index.js)로 인한 트리쉐이킹 방해 → named import로 전환
- 이미지 최적화 누락 (`next/image`, `astro:assets` 미사용)

### 청크 분리 기준

- 컴포넌트 파일 200줄 초과
- 특정 라우트에서만 사용되는 컴포넌트
- 초기 렌더에 불필요한 heavy 라이브러리

---

## STEP 4 — 캐시 정리

### 실행 순서

```bash
# 1. Next.js / Vite / Astro 빌드 캐시
rm -rf .next .turbo dist .astro

# 2. node_modules 캐시
npm cache clean --force

# 3. 패키지 재설치
npm install
```

---

## STEP 5 — 중복 코드 통합

### 탐지 기준

- 동일 로직이 2개 이상 파일에 존재
- 5줄 이상 유사한 JSX 블록
- 동일한 fetch / API 호출 패턴
- 같은 조건 분기 반복

### 통합 전략

| 중복 유형 | 통합 방법 |
|-----------|-----------|
| UI 블록 반복 | 컴포넌트 추출 → `src/components/` |
| 유틸 함수 반복 | 함수 추출 → `src/lib/` 또는 `src/utils/` |
| API 호출 반복 | 커스텀 훅 추출 → `src/hooks/` |
| 상수 반복 | 상수 파일 추출 → `src/constants/` |

### 프로젝트 컨벤션 준수

- named export only (default export 금지)
- barrel file 금지
- 한국어 주석
- ES2025+ 문법

---

## STEP 6 — 빌드 테스트

### 실행 순서

```bash
# 1. 타입 검사 (tsconfig 있는 경우)
npx tsc --noEmit

# 2. 린트 검사
npm run lint

# 3. 프로덕션 빌드
npm run build
```

### 결과 판정

| 상태 | 조건 | 다음 액션 |
|------|------|-----------|
| ✅ 통과 | 빌드 exit code 0 | 최종 보고 출력 |
| ❌ 실패 | 빌드 에러 발생 | 에러 분석 → 자동 수정 → 재빌드 |

### 자동 수정 재시도 규칙

1. 빌드 에러 로그 분석
2. 에러 파일·라인 특정
3. 수정 적용
4. `npm run build` 재실행
5. 3회 실패 시 수동 수정 필요 항목으로 보고하고 중단
